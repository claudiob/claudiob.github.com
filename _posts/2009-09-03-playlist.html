The playlist generator was the first project I created as part of my Ph.D in the area of music and recommender systems. The underlying idea was to collect a large amount of playlists from the web and reuse them to create new playlists that were meaningful from a musical point of view.

The code is written in lisp, a functional programming language that allows for backtracking, recursion and is particularly adapt for Case-Based Reasoning, the A.I. technique that drives the playlist generator.

The program looks in a large data set of playlists for patterns of successive songs that occur often, then assembles them to build new playlists with a given length and seed song. The mod-lisp component allows the programme to be executed remotely from any web browser. Yes, lisp is able to power advanced web applications!

Read the article I presented at the eccbr '06 for further details about CBR and playlist generation.

This code has been running between 2005 and 2008 on the Strands Labs web server. Unfortunately, this server has been dismantled, therefore the application cannot be tried online. You can download the source code and run the playlist generator either as a stand-alone application or as an Apache module.
